Two type of routing in web apps:
1 - Client side routing
2 - Server side routing

---- Understanding ----
We will understand:
What happens when app starts
How routing works
How Header works
How Body fetches data
How clicking restaurant works
How everything connects

🧠 STEP 1: When Your App Starts

Inside index.html you have:
<div id="root"></div>

In App.js:
const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<RouterProvider router={appRouter} />);

👉 This means:
React takes control of <div id="root">
And instead of rendering <AppLayout /> directly,
you render <RouterProvider />.

So now:
Router controls your whole app.

🧭 STEP 2: Understanding Routing (Very Important)
You created:
const appRouter = createBrowserRouter([
  {
    path: "/",
    element: <AppLayout />,
    children: [...]
  }
]);

This means:
When URL starts with /,
render AppLayout.

What is AppLayout?
const AppLayout = () => {
  return (
    <div>
      <Header />
      <Outlet />
    </div>
  );
};

Important:
<Header /> always shows
<Outlet /> changes depending on URL

Think like this:
Header = permanent
Outlet = changeable area

🏠 Example 1: When URL = "/"
Router checks:
{
  path: "/",
  element: <Body />
}

So inside Outlet, it renders:
Body component

Final screen:
Header
Body

📄 Example 2: When URL = "/about"
Router checks:
{
  path: "/about",
  element: <About />
}

Now inside Outlet:
About component
Final screen:
Header
About

Header never changes.
Only Outlet changes.
That is why Outlet is powerful.

🔗 STEP 3: How Navigation Works (Link)
In Header:
<Link to="/about">About</Link>
When you click:
React does NOT reload page ❌
It just changes component inside Outlet ✅
This is called:
Single Page Application (SPA)

🔘 STEP 4: Login Button Logic
const [btnNameReact, setbtnNameReact] = useState("Login");

When you click button:
btnNameReact === "Login"
  ? setbtnNameReact("Logout")
  : setbtnNameReact("Login");

What happens?
State changes
Header re-renders
Button text updates

Important:
If you used normal variable:
let btnName = "Login";

UI would NOT update.
Because React only tracks state.

🍽 STEP 5: Body Component (Most Important)

When Body loads:
useEffect(() => {
  fetchData();
}, []);

Empty array means:
Run only once.

Now what happens?

Step 1
Body renders first time.
listOfRestro = []
So:
listOfRestro.length === 0
TRUE
So Shimmer shows.

Step 2
fetchData runs:
const data = await fetch(...);
const json = await data.json();
fetch → gets response
json() → converts to JavaScript object

Step 3
You extract restaurants:
const restaurantData = json?.data?...?.restaurants;
Optional chaining prevents crash.

Step 4
You update state:
setListOfRestro(restaurantData);
setFilteredRestro(restaurantData);
Now React re-renders Body.

Step 5
Now listOfRestro is NOT empty.
So this runs:
filteredRestro.map(...)
Restaurant cards appear.
Shimmer disappears.

🔍 STEP 6: Search Logic
When typing:
value={searchText}
onChange={(e) => setSearchText(e.target.value)}
Input is controlled by React state.

When clicking Search:
listOfRestro.filter(...)
You filter original data.
Then:
setFilteredRestro(filteredRestro);
UI updates automatically.

⭐ STEP 7: Top Rated Button
listOfRestro.filter((res) => res.info.avgRating > 4.2)
You filter high rating restaurants.
Then update:
setFilteredRestro(filteredRestro);
UI updates.

🧾 STEP 8: Restaurant Card Click
In Body:
<Link to={"/restaurants/" + restaurant.info.id}>
Example:
If id = 123
URL becomes:
/restaurants/123
Router sees:
path: "/restaurants/:resid"
It matches.
Now Outlet renders:
RestroMenu component.
This is dynamic routing.
:resid is variable.

❗ STEP 9: Error Page
If you type wrong URL:
/random
Router cannot match.
So it shows:
errorElement: <Error />
Inside Error:
const err = useRouteError();
It displays error details.

🖼 STEP 10: RestroCard
Body sends data:
<RestroCard resData={restaurant} />
Inside RestroCard:
const {resData} = props;
Then:
const {name, avgRating} = resData.info;
It displays restaurant info.
This is props system.
Parent → Child data flow.

------- RestroMenu.js ------

import { useEffect, useState } from "react";
import Shimmer from "./Shimmer";
import { useParams } from "react-router-dom";
import { MENU_API } from "../Utils/constants";

const RestroMenu = () => {

    const [resInfo, setResInfo] = useState(null);
    const { resid } = useParams();

    useEffect(() => {
        fetchMenu();
    }, []);

    const fetchMenu = async () => {
        const data = await fetch(MENU_API + resid);
        const json = await data.json();
        setResInfo(json.data);
    };

    // ✅ Loading guard
    if (!resInfo) {
        return <Shimmer />;
    }

    // ✅ Restaurant Info
    const info = resInfo?.cards?.[2]?.card?.card?.info;
    const { name, cuisines, costForTwoMessage } = info || {};

    // ✅ Get Regular Cards Safely
    const regularCards = resInfo?.cards
        ?.find(card => card.groupedCard)
        ?.groupedCard?.cardGroupMap?.REGULAR?.cards;

    // ✅ Find section that contains itemCards
    const itemSection = regularCards?.find(
        c => c?.card?.card?.itemCards
    );

    const itemCards = itemSection?.card?.card?.itemCards;

    // ✅ If still no items
    if (!itemCards) {
        return <Shimmer />;
    }

    return (
        <div className="menu">
            <h1>{name}</h1>
            <p>{cuisines?.join(", ")}</p>
            <h3>{costForTwoMessage}</h3>

            <h2>Menu</h2>

            <ul>
                {itemCards.map((item) => (
                    <li key={item.card.info.id}>
                        {item.card.info.name} - Rs.
                        {(item.card.info.price ||
                          item.card.info.defaultPrice) / 100}
                    </li>
                ))}
            </ul>
        </div>
    );
};

export default RestroMenu;
